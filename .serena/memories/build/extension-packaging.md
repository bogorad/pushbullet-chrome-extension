`npm run build` now runs `scripts/build-extension.js`, which creates a complete unpacked extension under `dist/`. The script bundles the five TS entry points into `dist/*.js`, copies `manifest.json`, the four HTML entry pages, `css/`, and `icons/`, and rewrites built paths so `dist/manifest.json` points to `background.js` and built HTML files load `popup.js`/`options.js`/etc. directly from `dist` root. Regression coverage lives in `tests/scripts/build-extension.test.ts`. Browser verification showed the clean `dist/` package loads without the prior Chrome warnings about `.direnv` key files.
